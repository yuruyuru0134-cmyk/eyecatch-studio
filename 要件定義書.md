---

## 1. 目的・背景

- 市場で「画像生成」案件は **53件**確認でき、確実に収益化できる領域。
- 記事タイトルを入力するだけで、**Claude がプロンプトを組み立て → Gemini が画像生成**する半自動ワークフローを提供する。
- ブログ・メディアのアイキャッチ（16:9）制作を高速化することが狙い。

---

## 2. スコープ

### 2.1 本ツールが行うこと（AIに任せる領域）
1. 記事タイトルから **画像生成プロンプトの組み立て**（Claude）
2. **画像生成 API の呼び出し**（Gemini / Imagen）
3. 16:9 の画像を **3案** 出力し、選択・ダウンロードできる
4. **再生成（リロール）** ができる

### 2.2 人間が判断する領域（ツール対象外）
- 「どの画像なら使えるか」の最終的な採用判断
- 商用利用ルールの確認・適用判断

---

## 3. 機能要件

| ID | 機能 | 内容 |
|----|------|------|
| F-01 | タイトル入力 | 記事タイトルを入力するテキストフィールド |
| F-02 | プロンプト自動組み立て | Claude が入力タイトルから英語の画像生成プロンプトを生成（構造化出力） |
| F-03 | 画像生成（3案） | Gemini で **16:9・3案**を同時生成 |
| F-04 | 画像選択 | 3案からクリックで1枚を選択（ハイライト表示） |
| F-05 | ダウンロード | 各画像を個別にダウンロード可能 |
| F-06 | リロール | 「再生成」ボタンで同条件で再生成 |
| F-07 | 失敗時リトライ | エラー時に再試行できる（サーバー側でも1回自動リトライ） |
| F-08 | 権利保護スイッチ | 実在の人物・物の使用可否をチェックボックスで管理（後述） |
| F-09 | 生成内容の表示 | Claude が組み立てた「何を生成するか」の日本語サマリを表示 |

---

## 4. 権利保護要件（肖像権・著作権）★重要

> **方針: 現実の人物・物をそのまま使用することは原則「厳禁」。**

| ID | 要件 |
|----|------|
| R-01 | 実在の人物（有名人含む）、実在のブランドロゴ・商標、著作権で保護されたキャラクター・作品、特定可能な実在建造物などを **デフォルトで生成禁止**とする |
| R-02 | 許可の管理は **チェックボックスの ON/OFF** で行う |
| R-03 | **「許可しない」がデフォルト（= 不許可が ON の状態）** とする。チェックボックスは初期状態で「許可しない」側 |
| R-04 | 事前に許可された対象のみ、チェックボックスを「許可する」に切り替えて生成可能とする（任意で許可対象をメモ入力） |
| R-05 | 不許可時、Claude のシステムプロンプトに強い制約を注入し、抽象化・オリジナル表現に誘導する |
| R-06 | 許可時も、利用者自身の権利確認を前提とする旨を UI に明示する |

### 権利保護の動作仕様
```
[初期状態]
 ☑ 実在の人物・物を使用しない（推奨・デフォルトON）
   → Claudeが「実在人物/ロゴ/商標/著作物/特定建造物」を避け、
     架空・抽象・一般化した表現でプロンプトを構築

[許可に切替時]
 ☐ 実在の人物・物を使用しない（OFF = 許可）
   → 「許可済みの対象」入力欄が出現
   → 利用者の責任で権利処理済みの対象のみ使用する旨を明示
```

## 5. 達成基準（受け入れ条件）

| ID | 基準 | 検証方法 |
|----|------|----------|
| A-01 | 16:9 比率で出力される（実寸 **1280×720 以上**） | 生成画像の解像度確認（Imagen 4 = 1408×768 で基準クリア） |
| A-02 | プロンプト → 完成まで **30秒以内** | 計測（Claudeは低レイテンシ設定、画像3案は並列生成） |
| A-03 | 失敗時にリトライできる | エラー表示＋再試行ボタンの動作確認 |
| A-04 | 権利保護がデフォルトで有効 | 不許可時に実在対象が出ないことを確認 |

## 9. データフロー / 処理シーケンス

```
ユーザー入力（タイトル, 権利スイッチ）
        │
        ▼
[/api/generate]  ── Claude (claude-opus-4-8) ──▶ 構造化プロンプト
        │            (権利保護ルールを system prompt に注入)
        ▼
   Gemini Imagen (16:9 × 3案, 並列)
        │
        ▼
   base64画像3枚 + 日本語サマリ を返却
        │
        ▼
   フロントで表示・選択・DL・リロール
```

## 10. API 設計（概要）

### `POST /api/generate`
**Request**
```json
{
  "title": "記事タイトル",
  "allowRealEntities": false,
  "allowedNote": "（許可時のみ）許可済み対象のメモ"
}
```
**Response（成功）**
```json
{
  "summary": "日本語の生成内容サマリ",
  "promptUsed": "Imagenに渡した英語プロンプト",
  "images": ["data:image/...;base64,...", "...", "..."]
}
```
**Response（失敗）**
```json
{ "error": "エラーメッセージ" }
```

## 11. 制約・前提

- Claude / Gemini それぞれの **API キーが必要**（`.env.local` で管理。Git にはコミットしない）。
- Imagen は有料枠（API キーに Imagen 利用権限）が必要な場合がある。利用不可時は代替モデルへフォールバック（best-effort）。
- 生成画像の最終的な商用利用可否・権利確認は利用者の責任。
